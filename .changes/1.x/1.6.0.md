
## 1.6.0 - 2026-05-06

### Added

- new converter for Mago PHP toolchain (<https://github.com/carthage-software/mago>)
- all converters may now add response files on `invocations` when option is provided

### Changed

- add missing Region endColumn on Artifact Location
- add `tool.driver.version` for SARIF viewer tool that does not support the `tool.driver.semanticVersion` property
- all converters can now display rules that have "parameters" (see `defaultConfiguration`)
- allows to install PHPUnit major version 12 and 13
- migrate all CaptainHook config files to be compatible with `bartlett/captainhook-bin-plugin` version 1.1 or greater
- upgrade BOX Manifest to latest version (4.4) and add new Composer Tree text manifest copy
- ECS: `symplify/coding-standard` has moved base location of rule doc since v12.3.0

### Fixed

- custom autoloader: `autoload.php` (for additional vendor dependencies)
- resources builder: `resources/build.php` (especially for `clue/graph-composer` fork PHP 8.2 compatibility)
- `AbstractSource::getName()` that may return unexpected source name
- `Uri` on Artifact Location
- PHPMD normalizer: avoid invalid `helpUri` property; PHPMD 3.x Superglobals rule return invalid URI contents
- SARIF normalizer: for possible converters that produce insufficient inputs

**Full Changelog**: [1.5.0...1.6.0](https://github.com/llaville/sarif-php-converters/compare/1.5.0...1.6.0)
